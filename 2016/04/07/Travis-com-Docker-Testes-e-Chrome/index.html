Travis com Docker, Testes e Chrome

  • chrome
  • docker
  • tests
  • travis
Continuous Integrationtravis

Recentemente me deparei com alguns desafios e armadilhas para rodar os meus testes “End to End”, no ambiente sem GUI como o Docker, e a instância do próprio Travis.

O Browser é naturalmente necessário pra simular a interação do usuário no caso de testes e2e, mas também para os testes unitários, uma vez que no meu caso estava utilizando Karma localmente, configurado para rodar em browser e eu queria manter o ambiente homogêneo, ao do CI (Integração Contínua).

Em momento oportuno, gostaria de escrever com mais calma sobre a integração Travis, Docker, Mongo e Node, mas o desafio maior foi como rodar o Browser no ambiente Docker e Travis.

Alternativas

Por opção pessoal preferi partir de uma imagem ubuntu, como base no Dockerfile. Mas é bom saber, que existem algumas imagens que possuem Selenium e Chrome já pré configuradas pelo Docker Hub, porém eu não cheguei a testá-las. Se você optar por estas, talvez possa pular algumas configurações abaixo. Agora, se você é igual eu, que gosta de ter controle total “da sua imagem” (em qualquer sentido que queira interpretar), vamos ao passo a passo:

Java e Chrome

Deixarei aqui também, a maneira que instalei o Chrome e suas dependências:

Se tentar rodar agora, dará erro, pois o sistema tentará abrir o Browser (graficamente), e não vai conseguir.

X virtual framebuffer

X virtual framebuffer, ou Xvfb, é um display server que implementa o protocolo X11. Em contraste com outros display servers, Xvfb realiza todas as operações gráficas em memória sem apresentar qualquer saída de tela.

Ou seja, este é o ponto principal deste post. Graças ao Xvfb, será possível rodar o nosso Chrome que acabamos de instalar, em memória, sem necessidade de representação gráfica.

Travis

No .travis.yml, eu precisei fazer algumas configurações, as de exportar o Display, e o comando de chamar o Xvfb, eu precisei repetir no container e no ambiente travis.

# ...
export DISPLAY=:99.0
Xvfb :99.0 -extension RANDR > /dev/null &
docker exec -d -it meuContainer Xvfb :99.0 -extension RANDR
# Talvez você queira colocar alguma lógica pra saber quando o servidor Xvfb está de pé,
# pra poder seguir pra próxima linha, ou então usar o simples sleep como abaixo. Dez segundos estaria ótimo.
sleep 10
#...
